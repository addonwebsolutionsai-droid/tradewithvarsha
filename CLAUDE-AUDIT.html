2 · Current architecture problems

2.1 · The "Read ALL .md files" anti-pattern

CLAUDE.md line 3: "Read ALL linked .md files before modifying any code." This is the single biggest bloat. It forces Claude to load TECHSTACK.md + STRUCTURE.md + STRATEGIES.md + CONFIG.md + AGENTS.md + BACKTEST.md + ERRORS.md + INDICATORS.md + ASTRO.md + GANN.md + STRATEGIES_TURTLE_SOUP.md = 11 files, 80KB, before any change. Industry standard: lazy-load via @mentions. Only read the doc relevant to the task.

2.2 · Duplicated instruction files

files_full_sys/CLAUDE.md and hedge-fund/CLAUDE.md are byte-identical. Both get loaded because Claude Code walks up directories looking for CLAUDE.md and includes every one found. Fix: keep only the inner one, delete the outer.

2.3 · server/src/index.ts is 3,086 lines

This is a "god file" — Express setup, all routes, ALL cron schedulers, telegram dispatchers, lifecycle hooks. Touching it forces re-reading the whole thing. Industry standard:
  • src/routes/auth.ts
  • src/routes/picks.ts (weekly, daily)
  • src/routes/signals.ts
  • src/routes/admin.ts
  • src/routes/snapshots.ts
  • src/routes/lifecycle.ts
  • src/cron/index.ts (centralised scheduler — currently inline)
  • src/dispatchers/telegram.ts (currently inline ~600 LOC of dispatcher code in index.ts)
Each module 100-300 lines max. Targeted reads slash per-task tokens by 80%.

2.4 · 19 memory files with overlapping rules

feedback_quality_over_quantity, feedback_no_brainer_anchor, feedback_sl_tier_aware all encode "quality picks with proper SL/TP". feedback_pre_breakout_not_chasing and feedback_full_market_scan both encode the universe rule. Memory should be 7 atomic rules, not 19 cross-referenced ones.

2.5 · No typed interfaces — heavy use of any

(row as any).bucket = cand.bucket · const r = req.body ?? {} as any · etc. Type annotations make Claude read fewer files because it can infer shapes from the type. Adding interface PickRow, interface LifecycleEntry at module boundaries → fewer "what is this field?" lookups.

2.6 · Inline telegram dispatchers in index.ts

~600 LOC of dispatchXxxAlerts functions live inside index.ts. Should move to src/dispatchers/{weekly,daily,master,turtle,fib,harmonic,lifecycle}.ts. Each dispatcher = self-contained module.
